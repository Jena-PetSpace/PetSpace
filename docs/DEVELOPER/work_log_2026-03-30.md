## 2. Supabase SQL 보강

### 신규 RPC 함수 3개
- `confirm_kakao_user_by_email` — 카카오 로그인 시 이메일 자동 인증
- `get_popular_hashtags` — 인기 해시태그 조회
- `get_trending_hashtags` — 트렌딩 해시태그 조회

### RLS 보안 보강 2건
- `users` DELETE 정책 추가 (본인 계정만 삭제 가능)
- `notifications` DELETE 정책 추가 (본인 알림만 삭제 가능)

### handle_new_user 트리거 개선
- `ON CONFLICT (id) DO NOTHING` → `DO UPDATE SET` (provider, photo_url 갱신)
- 구글 이름 매핑: `full_name`, `name` 필드 fallback 추가
- 구글 사진 매핑: `avatar_url` 필드 fallback 추가

## 3. 인증 플로우 전면 재정비

### 이메일 인증 재활성화
- `emailConfirmedAt: DateTime.now()` 하드코딩 제거
- 로그인 시 인증 체크 재활성화
- SMTP 에러 무시 코드 제거 (Gmail SMTP 설정 완료)

### 비밀번호 재설정
- `AuthPasswordResetRequested` BLoC 핸들러 추가

### 구글 로그인 버그 수정
- email 기반 다른 사용자 프로필 반환 버그 제거
- `insert` → `upsert` 변경
- 트리거 프로필 생성 대기 500ms 추가

### 카카오 로그인 버그 수정
- RPC 파라미터명 `user_email` → `p_email` (SQL 함수와 일치)
- 사용자 취소 시 에러 메시지 숨김 (빈 메시지 반환)
- `insert` → `upsert` 변경

### 온보딩 프로필 덮어쓰기 방지
- `AuthOnboardingCompleted`에서 DB 최신 프로필 재조회
- 온보딩 중 ProfileService로 저장한 닉네임 유지
- 사진 미선택 시 카카오 프로필 사진 제거 (null 저장)

### 계정삭제 시 Storage 정리
- 5개 폴더 재귀 삭제: profiles, pets, posts, emotion_analysis, chat

### Kakao SDK 초기화 안전 처리
- try-catch 래핑 (초기화 실패 시 앱 크래시 방지)

## 4. Supabase 인프라 설정

### Gmail SMTP 설정
- `smtp.gmail.com:465` 연결
- Sender name: 멍냥다이어리
- 앱 비밀번호 인증

### Confirm email 활성화
- Authentication > Settings > Enable email confirmations ON

### Storage 보안
- 파일 크기 제한: 5MB
- MIME 타입: image/jpeg, image/png, image/webp

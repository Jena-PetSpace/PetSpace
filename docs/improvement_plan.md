# PetSpace 추가 개발 및 수정 계획

> 기준일: 2026-03-18
> 세 관점: 프론트엔드 10년차 / 백엔드 10년차 / 유저

---

## P0 — 출시 전 필수 (심사 리젝 방지)

### FE-P0-1: 네트워크 끊김 시 크래시 방어
- **문제**: 네트워크 없을 때 API 호출 시 무한 로딩 또는 크래시
- **해결**: 모든 BLoC에서 NetworkFailure 시 사용자 친화적 에러 메시지 표시
- **대상 파일**: FeedBloc, ProfileBloc, CommentBloc, HealthBloc

### FE-P0-2: 로딩 상태 통일 (Shimmer)
- **문제**: 일부 화면에서 데이터 로딩 시 빈 화면 노출
- **해결**: Shimmer skeleton 위젯 생성 후 피드/프로필/알림 화면에 적용

### BE-P0-1: RLS 정책 점검
- **문제**: 다른 유저의 데이터를 조회/수정할 수 있는 보안 허점 가능
- **해결**: 모든 테이블의 SELECT/INSERT/UPDATE/DELETE RLS 정책 확인

### BE-P0-2: notifications 테이블 인덱스
- **문제**: 알림 조회 시 user_id + created_at 인덱스 없으면 느려짐
- **해결**: 복합 인덱스 추가 SQL 실행

### UX-P0-1: 온보딩 가이드
- **문제**: 첫 실행 시 뭘 해야 하는지 모름
- **해결**: 반려동물 등록 유도 배너 + 간단한 3단계 온보딩

### UX-P0-2: 감정 분석 결과 한줄 요약
- **문제**: 수치만 보면 의미 파악 어려움
- **해결**: 결과 페이지 상단에 "오늘 행복해 보여요!" 같은 요약 문구

## P1 — 출시 직후 우선 개선

### FE-P1-1: 이미지 로딩 실패 placeholder
- CachedNetworkImage에 errorWidget 누락된 곳 전수 확인
- 기본 아이콘 또는 회색 placeholder 적용

### FE-P1-2: 키보드 overflow 방지
- TextField 있는 화면에서 키보드 올라올 때 overflow 발생 확인
- SingleChildScrollView 또는 resizeToAvoidBottomInset 적용

### BE-P1-1: API 호출 Rate Limiting
- 좋아요 연타, 댓글 도배 방지
- 클라이언트 debounce (300ms) 적용

### BE-P1-2: 이미지 업로드 크기 제한
- Storage에 10MB 이상 파일 업로드 방지
- 업로드 전 클라이언트에서 리사이즈 (max 1920px)

### UX-P1-1: 알림 배지 (빨간 점)
- 하단 탭바 알림 아이콘에 읽지 않은 알림 개수 배지 표시
- Realtime 구독으로 실시간 업데이트

### UX-P1-2: 기본 프로필 아바타
- 프로필 사진 없을 때 이름 이니셜 원형 아바타 표시

## P2 — 안정화 후 개선

### FE-P2-1: 하드코딩 색상 → AppTheme 토큰
- 다크모드에서 안 보이는 텍스트 방지
- 195개 하드코딩 색상 중 핵심 화면 우선 마이그레이션

### FE-P2-2: Empty State 일러스트
- 피드/알림/북마크/검색 결과 없을 때 안내 화면
- Lottie 또는 SVG 일러스트 + 행동 유도 버튼

### BE-P2-1: Supabase Storage RLS
- 다른 유저의 사진 삭제 가능 여부 확인
- Storage RLS 정책 추가

### BE-P2-2: 소프트 삭제
- 게시글/댓글 물리 삭제 → deleted_at 컬럼 추가
- 실수로 삭제 시 복구 가능

### UX-P2-1: 게시글 수정 기능
- 현재 삭제 후 재작성만 가능 → 수정 버튼 추가

### UX-P2-2: 앱 내 도움말/튜토리얼
- 감정 분석 사진 잘 찍는 팁
- 각 기능 설명 페이지

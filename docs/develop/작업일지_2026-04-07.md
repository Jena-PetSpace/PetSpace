## 2. 팔로우 버튼 무반응 수정

**파일**: `profile_page.dart`, `app_router.dart`, `search_page.dart`

`currentUserId`가 null이라 팔로우 이벤트가 즉시 return되던 문제.  
3단계 폴백으로 해결:

1. `search_page.dart` → 프로필 이동 시 URL에 `currentUserId` 포함
2. `app_router.dart` → URL 파라미터 없으면 `AuthBloc.state`에서 fallback
3. `profile_page.dart` → 그래도 없으면 `Supabase.instance.client.auth.currentUser?.id`

## 3. MY탭 팔로워/팔로잉 통계 미갱신 수정

**파일**: `lib/core/services/profile_service.dart`, `my_profile_header.dart`, `my_page.dart`, `main_navigation.dart`

### 문제 1 — 컬럼명 오류
`profile_service.dart`의 `getProfileStats()`에서 잘못된 컬럼 참조:
- `posts.user_id` → `posts.author_id`
- `follows` 테이블 `select('id')` → `select('follower_id')` / `select('following_id')` (`id` 컬럼 없음)

### 문제 2 — 탭 전환 시 갱신 안 됨
GoRouter ShellRoute로 캐시된 `MyPage`는 `initState`가 재호출 안 됨.  
`MyPageStatsNotifier` 싱글톤 추가:
- `MainNavigation`에서 MY탭(index 4) 탭 시 `MyPageStatsNotifier.instance.refresh()` 호출
- `MyPage`가 listener 등록 → `_statsRefreshKey` 증가
- `MyProfileHeader`가 `didUpdateWidget`으로 감지 → `_statsFuture` 재생성

## 8. 네이티브 스플래시 + 앱 시작 화면 통합

**파일**: `android/.../drawable/launch_background.xml`, `colors.xml`, `styles.xml`, `splash_page.dart`, `app_router.dart`

### 목표
앱 클릭 → 흰 배경 + 로고 (네이티브 스플래시) → 홈/로그인 바로 이동

### 변경 내용
- `launch_background.xml`: 흰 배경 + `splash_logo.png` (가운데)
- `colors.xml`: `splash_background` 색상 추가
- `assets/images/logo.png` → `android/.../drawable/splash_logo.png` 복사
- `app_router.dart`: `initialLocation` → `/home`, `AuthInitial`/`AuthLoading` 시 `/splash` redirect
- `SplashPage`: Lottie/애니메이션 제거, `BlocListener`로 인증 완료 시 직접 `context.go()` 처리

### 흐름
```
앱 클릭
  → 네이티브 스플래시 (흰배경 + 로고, 순식간)
  → Flutter 엔진 초기화 (AuthInitial → /splash 빈화면 유지)
  → AuthBloc 인증 확인 완료
  → BlocListener: AuthAuthenticated → /home
                  AuthUnauthenticated → /onboarding
```
